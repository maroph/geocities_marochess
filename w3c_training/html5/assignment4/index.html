Subject 3: for people with some JavaScript knowledge, about IndexedDB

Start from the examples provided in the IndexedDB book and adapt them in order to manage a database of the HTML5 interactive examples provided in this course. Objects stores in the datastore may be composed of, for example:

  1. the jsbin.com URL of the example,
  2. a description,
  3. the name of the chapter in which the example has been presented, and,
  4. the name of the section in the chapter.
  5. Eventually, a screenshot for the example (URL, or if you are a bit on the geeks side, image content)

You will provide an html page that contains your application. When loaded it must populate the indexedDB database with at least 10 different examples, display them in a table or in a list. The GUI must also provide a search form for getting all examples from a given chapter. We assume that the application will run with Chrome's latest version.

    (complete source of the form sample)

Description

The assignment page offers for options:

  1. Create Database
    This option will create and fill the IndexDB
  2. Show All Snippets
    This option shows a table of all code snippets in the IndexDB
  3. Show All Snippets of a Week
    This option shows a table of all code snippets of a certain course week (1-6) in the IndexDB
  4. Delete Database
    This option will delete the IndexDB

I haven't coded any input data check. So please use it with caution. Also I simply show the desired data without any sorting. I have added one snapshot only to show how I have intended to handle the snapshot images: I simply use data URLs (= MIME coded image data). This seems to me the easiest (but not the best) way to implement this feature.

All in all I can say that I don't like the IndexedDB API because everything is asynchronous but the web page usage itself is synchronous! Seems much to complex to use for me. If necessary I will use localStorage/sessionStorage and for bigger data I will use a server-side database.
