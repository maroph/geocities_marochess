Extend the MyDiary Application

A few things that would greatly enhance the Diary Application that you created in Assignment 2 are:

  • A simple form to actually create new entries through a form!
  • The ability to manage (and delete) existing entries.
  • The ability to log where they user was were when they wrote a given entry.

Modify your Assignment 2 diary application so that the user can actually manage their own entries. When the user saves an entry, add the location information for each entry and store the entry data into localStorage. Be sure to take into account the fact that the user may choose not to provide Geolocation information and that the APIs may not be available.

OPTIONAL: If you are feeling like challenging yourself, add swipe event support as a means to delete an entry (or find some other interesting use for it). Again, be sure to take a progressive enhancement approach here. You might also like to add support for taking a photo if the browser supports it (covered in Week 5) or make use of any of the bleeding edge APIs covered in Week 5.

Assignment 3: My Assignment 3 Data

I viewed the web app with the following devices:

  • Samsung Galaxy S4 (Android 4.2)
  • Kindle Fire HD
  • Apple iPad Air (iOS 7)
  • Windows 7 PC 24inch monitor with the current Chrome and Firefox versions

You can call the MyDiary web application via two URLs:

  • diary/index.html
    (MyDiary)
  • diary/index.html?clear
    (MyDiary with optional all entries removal - if entries are available)

I have implemented the following features:

  • Add a new entry
  • Modify an existing entry
  • Delete an entry
  • Optional adding of location info to an entry
  • View the location data on a Google map
  • Optional adding an image/photo an entry
  • View the image in an extra window
  • Show hints via a title attribute for almost all actions
    (useful for mouse based devices)

By default you will see only the head lines of the entries. By clicking on each headline you can expand/collapse the related entry. As an alternative you can use the "Expand Entries" button to expand/collapse all entries at once.
All data are stored in one localStorage item as a stringified JSON array of entry objects (self defined JavaScript object DiaryEntry).

Create/modify an entry
You will see the following icons:

  • Save entry.
  • Cancel creation or modifying of an entry.
  • Add geo location data of current location.
  • Use camera for a snapshot.
    Opens a new tab where you can use the camera for a snapshot. After clicking on close or closing the tab manually you are back and tge photo will be saved with the entry.
  • Upload an image.
    Opens a new tab where you can upload an image or use the camera for a snapshot. After clicking on close or closing the tab manually you are back and tge photo will be saved with the entry.

Show an entry
You will see the following icons:

  • Modify the entry.
  • Delete the entry.
  • View the geo location data on a Google map.
  • View the stored image.

The snapshot functionality (./diary/camera.html) and the upload feature (./diary/imageLoader.html) can be used as part of the MyDiary web application or standalone.
A localStorage item ist used for the communication between the MyDiary web application and the above extra pages.

Assignment 3: Restrictions

I haven't implement the following two features

  • geo location data can be modified but not removed from an entry
  • image data can be modified but not removed from an entry

getUserMedia API Restrictions (./diary/camera.html)

  • PC with most recent Firefox and/or Chrome: works
  • Kindle Fire HD: won't work
  • Galaxy S4 with Chrome: works
  • iPad Air with default Safari: won't work

"input type=file caption=camera" Restrictions (./diary/imageLoader.html)

  • PC with most recent Firefox and/or Chrome
    Upload of images works, camera will not be used
  • Kindle Fire HD: won't work
    Upload of images works, camera will not be used
  • Galaxy S4 with Chrome: works
    Camera will be used
  • iPad Air with default Safari: works
    You can choose between camera usage and image upload
