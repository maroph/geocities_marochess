Assignment 1: Create an "About" page

For this first assignment, please write and publish a page that presents yourself with informations such as name, address, etc. Please add a picture as well.

The constraints are :

  1. Use as much as you can the new structuring elements (header, footer, article, section, summary, details, etc.) that you've seen in the first and second parts of this first week's content material
    • Add some CSS rules allowing you to "guess" the underlying structure of your page - by moving the mouse cursor. Read the blog example from the course to get some help.
  2. Use as many microdata as you can. We recommend that you read all the "external resources" provided in the microdata section of this week's content material. Particularly, please read the related chapter of Mark Pilgrim's book (http://diveintohtml5.info/extensibility.html). Even though this book needs some update (in particular, URLs used to reference vocabularies should be updated), it is one of the best resources out there about Microdata.
  3. Please test your page on different testing tools and provide screenshots of the different results.
  4. Finally, look for browser's extensions that recognize the different new elements seen in this chapter, and give a list of what you have found, you will find it interesting. Please provide links and screenshots.

All results must be online, and please regroup the links in one single page. To submit a response to this assignment, just provide the URL of this page.

My Findings

  • Support of details and summary elements: Only in current Chrome 25 and Safari 6 versions
  • Support of the img element download attribute: Only in current Chrome 25 version
  • Support of form input type range: Not in current Firefox 19
  • Chrome 25.0.1364.172 doesn't support document.getItems(...)
  • The download attribute in an a element is not part of the HTML5 W3C Working Draft 11 October 2012
  • The W3C Validator doesn't accept the download attribute in an a element
  • Brand new microdata usage example: The Send to Kindle Button for Websites
