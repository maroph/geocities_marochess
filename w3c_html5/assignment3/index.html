Assignment 3: Forms and geolocation

Provide an HTML5 form for registering in an imaginary forum. The form proposes to enter first name, last name, email, phone, address, the password (to enter twice and to validate using the form validation api), etc. Please use as many features you want decsribed in the HTML5 Forms chapter.

You will also propose a "locate me" button that will use the geolocation API, look at the example that uses the google geocoder. You should be able to fill automatically the input fields for the address once you are geolocalised.

As an extra exercise (facultative), you may also propose to add a picture to the form (using file input or drag'n'drop or Webcam input would be great!).

Description

My form is based on the sample given at http://webdesign.tutsplus.com/tutorials/site-elements/bring-your-forms-up-to-date-with-css3-and-html5-validation/.

The required fields are marked. I use input pattern for the following fields: Username, Password, Website, and Phone.

Clicking the "Locate Me" button will fill (if available) the fields City, ZIP, Street, and Country.

You can add an optional picture with a maximal width of 160 pixels and a maximal height of 120 pixels. The picture can be added either via drag & drop or via a webcam.

Drag & drop adding: the pictures are always drawn at the upper left corner of the area. If the picture is bigger than the given area (160x120) you can click with the mouse in the drop area and move the picture as long as the mouse is pressed.

Moving the picture after drag & drop works well with JPEG images but shows strange effects if you use GIF or PNG pictures :-(

Tested Browsers

I have tested my form with Firefox 20.0.1, the current Chrome 26.x version and Opera 12.15.
